# robots/

This directory contains robot sovereignty and safety boundary specifications.

Robots are embodied AI systems with physical-world capabilities. Their sovereignty framework requires additional care because their actions can cause physical harm. This directory will hold:

- Robot authority envelope patterns
- Physical domain specification guidance
- Fail-safe design patterns
- Emergency stop authority frameworks
- Multi-principal environment guidance for shared physical spaces

See [schemas/robot-authority.schema.json](../schemas/robot-authority.schema.json) for the base authority schema.
See [docs/robot-sovereignty.md](../docs/robot-sovereignty.md) for the doctrine.

**Note:** This directory does not and will not contain robot control software, actuator commands, specific hardware configurations, or production safety systems.
