# AGENTROPOLIS-SOVEREIGNTY

> **AI sovereignty is the right and capacity of individuals, institutions, agents, robots, and businesses to control their data, memory, models, workflows, permissions, identity, value exchange, and execution environments.**
>
> **Sovereignty is not isolation. Sovereignty is controlled interoperability.**

---

## What Is This?

AGENTROPOLIS-SOVEREIGNTY is an open, vendor-neutral doctrine repository defining AI-native sovereignty for:

- **Individuals** — humans controlling their own data, identity, memory, and AI interactions
- **Organizations** — institutions governing their AI systems, data flows, and agent authority
- **Agents** — autonomous AI software entities with bounded authority, accountability, and memory
- **Robots** — embodied AI systems with physical-world authority limits and safety boundaries
- **B2B systems** — business-to-business workflows with verifiable permissions and audit trails
- **B2AI infrastructure** — business-to-AI integrations with governed access and interoperability contracts

This repository contains doctrine documents, JSON schemas, governance principles, and examples. It is intentionally **implementation-light** — focused on concepts, structures, and principles rather than runnable code or production systems.

## Core Principles

1. **Sovereignty by default** — every entity starts with full control over its own domain; access is granted, never assumed.
2. **Controlled interoperability** — entities may choose to interoperate, but interoperability does not dissolve sovereignty.
3. **Accountability chains** — every action by an agent, robot, or automated system traces back to a sovereign principal.
4. **Memory ownership** — the entity that generates memory owns it; transfer requires explicit consent and audit.
5. **Permission minimalism** — permissions are specific, time-bounded, and revocable; no standing unlimited grants.
6. **Public safety boundary** — sovereign rights do not extend to actions that cause harm to others or circumvent lawful governance.
7. **Vendor neutrality** — this doctrine is not tied to any platform, cloud provider, AI vendor, or identity system.
